安装方式（取决于是否签名）

情况 A：已签名的 xpi（推荐，支持自动更新）

  1. 下载 .xpi 文件后，直接双击打开
  2. Firefox 弹出安装确认框，点击「添加」
  3. 安装完成，后续 Firefox 会自动检查 updates.json 更新

情况 B：未签名的 xpi（仅临时调试，重启后失效）

  1. 地址栏输入 about:debugging#/runtime/this-firefox
  2. 点击「加载临时附加组件」
  3. 选择下载的 .xpi 文件（或 manifest.json）
临时加载的局限：重启 Firefox 后扩展消失，且不会自动更新。 如需永久安装，开发者必须用 web-ext sign 对 xpi 签名（见下方开发者说明）。
✅ 安装成功标志：Firefox 工具栏出现 JIRA 效率助手图标。 点击它可打开模板管理页。

常见问题

Q：为什么 Firefox 提示"此附加组件无法安装，因其未被验证"？

A：这是 Firefox 的强制签名策略。未签名 xpi 只能用 about:debugging 临时加载。要永久安装，开发者需通过 AMO 签名（见 docs/RELEASE-GUIDE.md）。

Q：Firefox 安装后能自动更新吗？

A：只有签名过的 xpi 才能自动更新。Firefox 会检查 manifest 里的 update_url（指向 updates.json），发现新版自动下载安装。未签名版本无法自动更新。

Q：能否绕过 Firefox 的签名要求？

A：有两种非正式方式（不推荐）：
① Firefox 开发版/Nightly：设置 about:config → xpinstall.signatures.required = false
② Firefox Developer Edition：同样可关闭签名检查
正式版 Firefox 无法关闭签名检查。

Q：临时加载的扩展会保留数据吗？

A：会。扩展数据存在 browser.storage.local，即使临时加载失效，数据仍保留。重新加载后数据恢复。但 Firefox 卸载扩展时会清除数据。

Q：Firefox 版和 Chrome 版功能一样吗？

A：完全一样。所有业务代码（contentScript、background 逻辑、Vue 页面）两端共用，仅 manifest 和 background 入口方式不同（Chrome 用 service_worker，Firefox 用 background.scripts）。
